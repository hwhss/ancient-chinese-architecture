# 后端接口调整清单（支持前端 three.js LOD 策略）

## 问题概述

前端 three.js viewer 已实现完整的 LOD 加载策略（`loadLodPipeline()`），但后端接口返回的 `scene.lod[]` 数组中的 URL 尚未签名，存在以下问题：

1. **安全问题**：前端收到裸 URL，无法通过流量检查或授权验证
2. **失效问题**：长时间会话中，LOD 版本的 URL 可能无令牌续期机制，导致加载失败
3. **预加载问题**：`preload.lod0ModelUrl` 返回的是裸 URL，需要签名化

## 需要调整的接口

### 1. `GET /api/buildings/:id/model3d/manifest` 的 `scene.lod[]` 字段

**当前状态**：

```js
scene: {
  entry: "https://server/api/assets/signed?token=...",  // ✅ 已签名
  lod: [
    { level: 0, modelUrl: "https://cdn.com/v1-lite.glb", ... },   // ❌ 裸 URL
    { level: 1, modelUrl: "https://cdn.com/v1-hd.glb", ... }      // ❌ 裸 URL
  ]
}
```

**需要调整为**：

```js
scene: {
  entry: "https://server/api/assets/signed?token=...",  // ✅ 已签名
  lod: [
    {
      level: 0,
      modelUrl: "https://server/api/assets/signed?token=...",  // ✅ 签名
      signedUrl: "https://server/api/assets/signed?token=...",  // 或加这个字段
      expiresAt: "2026-03-27T10:30:00Z"
    },
    {
      level: 1,
      modelUrl: "https://server/api/assets/signed?token=...",  // ✅ 签名
      signedUrl: "https://server/api/assets/signed?token=...",  // 或加这个字段
      expiresAt: "2026-03-27T10:30:00Z"
    }
  ]
}
```

**实现位置**：`backend/src/services/buildingService.js` 的 `getBuildingModel3dManifest()` 函数

**建议方案**：

- 在 `buildSignedVersion()` 中新增参数 `signAllLod=true`
- 对 `lod` 数组中每个 item 的 `modelUrl` 执行签名
- 返回时用 `signedUrl` 替换或补充 `modelUrl`

---

### 2. `preload.lod0ModelUrl` 签名化

**当前状态**：

```js
preload: {
  placeholderType: "lod0-model",
  lod0ModelUrl: "https://cdn.com/model-lod0.glb",  // ❌ 裸 URL
  panoramaUrl: "https://cdn.com/panorama.jpg",
}
```

**需要调整为**：

```js
preload: {
  placeholderType: "lod0-model",
  lod0ModelUrl: "https://server/api/assets/signed?token=...",  // ✅ 签名
  lod0ExpiresAt: "2026-03-27T10:30:00Z",
  panoramaUrl: "https://cdn.com/panorama.jpg",  // 全景通常为公开资源，可不签名
}
```

**实现位置**：`backend/src/services/buildingService.js` 的 `buildPreloadInfo()` 函数

---

### 3. 支持多版本的 token 续期

**当前状态**：

- `GET /api/assets/refresh?token=xxx` 依据老 token 中的 `version` 字段续期
- 仅对主版本有效

**需要调整为**：

- 支持在 token 中指定 LOD level 标识
- 前端可为不同 LOD 版本提交续期请求
- 后端返回新签名的 LOD URL

**实现位置**：`backend/src/utils/signedAsset.js` 和 `backend/src/controllers/assetController.js`

**建议**：在 token payload 中加入可选的 `lodLevel` 字段，例如：

```js
{
  resourceUrl: "https://cdn.com/model.glb",
  buildingId: "gugong_01",
  version: "v1-hd",
  lodLevel: 1,  // ← 新增
  userId: "user123",
  role: "viewer",
  exp: 1711500600
}
```

## 优先级排序

| 优先级  | 任务                              | 难度 | 估算工时 | 备注           |
| ------- | --------------------------------- | ---- | -------- | -------------- |
| 🔴 最高 | scene.lod[] 签名化                | 低   | 30 分钟  | 前端已就绪等待 |
| 🔴 最高 | preload.lod0ModelUrl 签名化       | 低   | 15 分钟  | 前端已就绪等待 |
| 🟡 中   | token payload 加 lodLevel         | 中   | 45 分钟  | 长会话续期优化 |
| 🟢 低   | /api/assets/refresh 支持 LOD 续期 | 中   | 60 分钟  | 后续迭代优化   |

## 实现步骤（优先做最高优先级）

### Step 1：LOD 签名工厂函数

在 `backend/src/services/buildingService.js` 中添加：

```javascript
function buildSignedLodArray(lodArray, buildingId, version, requester) {
  if (!Array.isArray(lodArray)) {
    return [];
  }

  return lodArray.map((item) => {
    if (!item || !item.modelUrl) {
      return item;
    }

    const safeRequester = normalizeRequester(requester);
    const signed = buildSignedAssetUrl({
      resourceUrl: item.modelUrl,
      buildingId,
      version: `${version}-lod${item.level || 0}`, // e.g. "v1-hd-lod0"
      userId: safeRequester.userId,
      role: safeRequester.role,
      bindIp: safeRequester.ip,
      userAgent: safeRequester.userAgent,
      expireSeconds: config.assetSignExpireSeconds,
      secret: config.assetSignSecret,
      publicBaseUrl: config.publicBaseUrl,
    });

    return {
      ...item,
      modelUrl: signed.signedUrl, // ← 用签名 URL 替换原 URL
      signedUrl: signed.signedUrl, // ← 或额外提供这个字段
      expiresAt: signed.expiresAt, // ← 新增过期时间
    };
  });
}
```

### Step 2：修改 `getBuildingModel3dManifest()` return 语句

找到这一段：

```js
scene: defaultVersion ? {
  format: defaultVersion.format,
  entry: defaultVersion.signedUrl,
  environment: 'neutral',
  draco: defaultVersion.draco,
  ktx2: defaultVersion.ktx2,
  lod: defaultVersion.lod,  // ← 原来这样，需要签名
  expiresAt: defaultVersion.expiresAt
} : {...}
```

改为：

```js
scene: defaultVersion ? {
  format: defaultVersion.format,
  entry: defaultVersion.signedUrl,
  environment: 'neutral',
  draco: defaultVersion.draco,
  ktx2: defaultVersion.ktx2,
  lod: buildSignedLodArray(
    defaultVersion.lod,
    id,
    defaultVersion.version,
    requester
  ),  // ← 签名处理
  expiresAt: defaultVersion.expiresAt
} : {...}
```

### Step 3：修改 `buildPreloadInfo()` 函数

添加 LOD0 签名逻辑：

```javascript
function buildPreloadInfo(model3d, defaultVersion, buildingId, requester) {
  const preloadFromModel = model3d && model3d.preload ? model3d.preload : {};
  const preloadFromVersion =
    defaultVersion && defaultVersion.preload ? defaultVersion.preload : {};

  const lod0Item =
    defaultVersion && Array.isArray(defaultVersion.lod)
      ? defaultVersion.lod.find(
          (item) =>
            item && Number(item.level) === 0 && (item.modelUrl || item.url),
        )
      : null;

  let lod0ModelUrl =
    preloadFromVersion.lod0ModelUrl ||
    preloadFromModel.lod0ModelUrl ||
    (lod0Item ? lod0Item.modelUrl || lod0Item.url : "");

  let lod0ExpiresAt = "";

  // 对 LOD0 URL 签名
  if (lod0ModelUrl && defaultVersion) {
    const safeRequester = normalizeRequester(requester || {});
    const signed = buildSignedAssetUrl({
      resourceUrl: lod0ModelUrl,
      buildingId,
      version: `${defaultVersion.version}-lod0`,
      userId: safeRequester.userId,
      role: safeRequester.role,
      bindIp: safeRequester.ip,
      userAgent: safeRequester.userAgent,
      expireSeconds: config.assetSignExpireSeconds,
      secret: config.assetSignSecret,
      publicBaseUrl: config.publicBaseUrl,
    });
    lod0ModelUrl = signed.signedUrl;
    lod0ExpiresAt = signed.expiresAt;
  }

  const panoramaUrl =
    preloadFromVersion.panoramaUrl || preloadFromModel.panoramaUrl || "";

  const placeholderType = lod0ModelUrl
    ? "lod0-model"
    : panoramaUrl
      ? "panorama"
      : "poster";

  return {
    placeholderType,
    lod0ModelUrl,
    lod0ExpiresAt, // ← 新增
    panoramaUrl,
    posterBlur:
      preloadFromVersion.posterBlur || preloadFromModel.posterBlur || "",
    swapMode:
      preloadFromVersion.swapMode ||
      preloadFromModel.swapMode ||
      "replace-on-ready",
    uiHint:
      preloadFromVersion.uiHint ||
      preloadFromModel.uiHint ||
      "先展示轻量占位资源，再无缝切换高清模型",
  };
}
```

### Step 4：更新 `getBuildingModel3dManifest()` 中的 preload 调用

修改：

```js
const preload = buildPreloadInfo(profile.model3d || null, defaultVersion);
```

为：

```js
const preload = buildPreloadInfo(
  profile.model3d || null,
  defaultVersion,
  id,
  requester,
);
```

## 反向兼容性

当前调整是向后兼容的：

- 前端 viewer 可同时处理 `signedUrl` 和 `modelUrl` 字段
- 若返回的是裸 URL（老版本），前端会直接使用（不安全但可用）
- 若返回的是签名 URL，前端优先使用签名 URL

建议消息：`preload.lod0ModelUrl` 从"可能是裸 URL"升级为"必定是签名 URL"
