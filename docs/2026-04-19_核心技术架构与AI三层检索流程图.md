# 核心技术架构与AI三层检索流程图

下面两张图分别对应：

1. 前端到后端的核心技术架构流程
2. AI 问答的三层检索与兜底流程

## 1. 核心技术架构流程图

```mermaid
graph TB
    FE[前端页面\npages/index/index.vue\npages/map/map.vue\npages/detail/detail.vue] -->|发起 API 请求| APP[Express 应用层\nbackend/src/app.js]

    APP -->|CORS / JSON 解析 / 静态资源| ROUTER[路由层\nbackend/src/routes/index.js]

    ROUTER -->|/api/chat| CHAT_CTRL[chatController]
    ROUTER -->|/api/buildings\n/api/material\n/api/knowledge| COMMON_CTRL[业务控制器]
    ROUTER -->|/api/assets / /api/qiniu| ASSET_CTRL[资源控制器]

    CHAT_CTRL --> CHAT_SVC[chatService]
    COMMON_CTRL --> BIZ_SVC[业务 Service]
    ASSET_CTRL --> ASSET_SVC[asset / qiniu Service]
```

## 2. AI 问答三层检索流程图

```mermaid
graph TB
    Q[用户问题] --> NORM[问题预处理\nnormalizeQuestion]
    NORM --> TRY1[第一层\n知识库直接匹配]

    TRY1 --> KB1[searchKeywordCandidates\n从知识库候选中找强匹配]
    KB1 --> DEC1{是否命中并通过规则?}

    DEC1 -->|是| R1[返回知识库答案]
    R1 --> REWRITE[可选：rewriteKnowledgeAnswer]
    REWRITE --> ENTITY1[resolveBuildingEntities\n答案后实体对齐]
    ENTITY1 --> OUT1[输出 answer / source / materialId / entities]

    DEC1 -->|否| TRY2[第二层\n混合打分检索]

    TRY2 --> KW[关键词候选检索]
    TRY2 --> VEC[向量候选检索\nsearchVectorCandidates]
    KW --> MERGE[合并候选\nmergeCandidates]
    VEC --> MERGE
    MERGE --> SCORE[评分与决策\ndecisionScore / gap / 阈值]
    SCORE --> DEC2{是否接受混合召回?}

    DEC2 -->|是| R2[返回混合检索答案]
    R2 --> REWRITE2[可选：rewriteKnowledgeAnswer]
    REWRITE2 --> ENTITY2[resolveBuildingEntities\n答案后实体对齐]
    ENTITY2 --> OUT2[输出 answer / source / materialId / entities]

    DEC2 -->|否| TRY3[第三层\nAI fallback 兜底]

    TRY3 --> AI[getAIAnswer]
    AI --> ENTITY3[resolveBuildingEntities\n对 AI 输出做实体对齐]
    ENTITY3 --> OUT3[输出 answer / source=ai / materialId / entities]

    classDef input fill:#d3f9d8,stroke:#2f9e44,color:#14532d;
    classDef rule fill:#ffe3e3,stroke:#c92a2a,color:#7f1d1d;
    classDef process fill:#e5dbff,stroke:#5f3dc4,color:#3b1d74;
    classDef data fill:#fff4e6,stroke:#e67700,color:#8a4f00;
    classDef output fill:#c5f6fa,stroke:#0c8599,color:#005f6b;

    class Q,NORM input;
    class TRY1,DEC1,TRY2,DEC2,TRY3 rule;
    class KB1,KW,VEC,MERGE,SCORE,REWRITE,REWRITE2,ENTITY1,ENTITY2,ENTITY3,AI process;
    class R1,R2,OUT1,OUT2,OUT3 output;
```

## 说明

这两张图按当前后端实现整理：入口是 Express，路由进入 controller，再下沉到 service，数据访问落到 repository 或向量检索；AI 问答则按“知识库优先、混合检索、AI 兜底”三层展开。
