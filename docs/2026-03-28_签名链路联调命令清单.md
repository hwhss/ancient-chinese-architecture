# 签名链路联调命令清单（PowerShell）

## 0) 前置说明

1. 后端默认地址：`http://localhost:9527`
2. 本清单用于验证：
   - manifest 是否仍泄露裸 URL
   - scene/lod/preload 是否均为签名地址
   - refresh 是否返回合法签名地址

## 1) 拉取 Manifest 并查看核心字段

```powershell
$base = 'http://localhost:9527'
$resp = Invoke-RestMethod -Uri "$base/api/buildings/taihe_dian/model3d/manifest" -Method Get
$m = $resp.data

[PSCustomObject]@{
  sceneEntry = $m.scene.entry
  preloadLod0 = $m.preload.lod0ModelUrl
  versionCount = @($m.versions).Count
  lodCount = @($m.scene.lod).Count
} | Format-List
```

预期：

1. `sceneEntry` 包含 `/api/assets/signed?token=`
2. `preloadLod0` 包含 `/api/assets/signed?token=`

## 2) 检查 versions[] 是否暴露裸 modelUrl

```powershell
$keys = ($m.versions[0] | Get-Member -MemberType NoteProperty | Select-Object -ExpandProperty Name)
$keys -join ', '
```

预期：

1. 不应包含 `modelUrl`
2. 最小字段建议：`version,label,format,signedUrl,expiresAt,draco,ktx2`

## 3) 检查 scene.lod[] 是否全签名

```powershell
$allSigned = $true
foreach ($lod in @($m.scene.lod)) {
  $u = [string]$lod.modelUrl
  if (-not ($u -match '/api/assets/signed\?token=')) {
    $allSigned = $false
  }
}
$allSigned
```

预期：

1. 输出 `True`

## 4) 检查 refresh 行为

```powershell
$token = ([uri]$m.scene.entry).Query -replace '^\?token=', ''
$refreshResp = Invoke-RestMethod -Uri "$base/api/assets/refresh?token=$token" -Method Get
$rd = $refreshResp.data

[PSCustomObject]@{
  shouldRefresh = $rd.shouldRefresh
  secondsLeft = $rd.secondsLeft
  signedUrlPrefixOk = ([string]$rd.signedUrl -match '/api/assets/signed\?token=')
  expiresAt = $rd.expiresAt
} | Format-List
```

预期：

1. `signedUrlPrefixOk = True`
2. `shouldRefresh` 在窗口外可能是 `False`，窗口内为 `True`

## 5) 模拟反向代理域名（验证不写死 localhost）

```powershell
$headers = @{
  Host = 'api.example.com'
  'X-Forwarded-Proto' = 'https'
}
$resp2 = Invoke-RestMethod -Uri 'http://localhost:9527/api/buildings/gugong_01/model3d/manifest' -Headers $headers -Method Get
$resp2.data.scene.entry
```

预期：

1. 返回应以 `https://api.example.com/` 开头
2. 若仍是 `http://localhost:9527/`，说明仍有写死风险

## 6) 若结果与预期不一致，先做这两步

1. 确认后端进程已重启到最新代码。
2. 确认运行的是 `backend/app.js`（不是历史旧进程或其他端口实例）。
