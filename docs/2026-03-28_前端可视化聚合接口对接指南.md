# 2026-03-28 前端可视化聚合接口对接指南

## 1. 文档目的

本文件用于给前端同学直接对接后端可视化接口。

适用场景：

1. 首页可视化看板。
2. 地图点位渲染。
3. 时间轴展示。
4. 统计图（柱状图、饼图、排行）。

统一响应格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

## 2. 环境与基础信息

1. 本地后端地址: http://localhost:9527
2. 健康检查: GET /api/health
3. 可视化接口统一前缀: /api/visualization

## 3. 接口清单

1. GET /api/visualization/overview
2. GET /api/visualization/map-points
3. GET /api/visualization/timeline
4. GET /api/visualization/stats

## 4. 接口说明

### 4.1 总览接口

- 方法: GET
- 路径: /api/visualization/overview
- 用途: 首页看板统计

示例响应（data）：

```json
{
  "summary": {
    "totalBuildings": 17,
    "withCoordinates": 17,
    "with3d": 2,
    "coverageRatio": 1
  },
  "distributions": {
    "byCategory": [{ "key": "palace", "count": 3 }],
    "byRegion": [{ "key": "北京市", "count": 3 }],
    "byHeritageLevel": [{ "key": "national", "count": 17 }],
    "byEraBucket": [{ "key": "明", "count": 4 }]
```

```json
}
```

### 4.3 时间轴接口

- 方法: GET
- 路径: /api/visualization/timeline
- 用途: 时间轴与年代分布

可选查询参数：

1. category: palace|bridge|garden|defense|residence|tower|water|all
2. region: 省级区域名
3. city: 城市名
4. eraStart: 年代起点（整数）
5. eraEnd: 年代终点（整数）

```text
/api/visualization/timeline?category=palace&region=北京市
```

### 4.4 统计接口

- 方法: GET
- 路径: /api/visualization/stats
- 用途: 通用统计图表

可选查询参数：

1. dimension: category|era|region|heritageLevel|openStatus（默认 category）
2. category: palace|bridge|garden|defense|residence|tower|water|all
3. region: 省级区域名
4. city: 城市名
5. eraStart: 年代起点（整数）
6. eraEnd: 年代终点（整数）

示例请求：

```text
/api/visualization/stats?dimension=region&category=all
```

示例响应（data）：

```json
{
  "filters": {
    "category": "all",
    "region": "",
    "city": "",
    "eraStart": null,
    "eraEnd": null
  },
  "dimension": "region",

```

## 5. 前端推荐调用顺序

### 5.1 首页看板

1. 首屏并行请求 overview 与 stats。
2. overview 填充总览卡片。
3. stats 作为图表数据源。

### 5.2 地图页

1. 请求 map-points。
2. 用 points 渲染地图标记。
3. 用 meta 显示数据完整性（例如缺失坐标数量）。

### 5.3 时间轴页

1. 先按筛选条件请求 timeline。
2. timeline 用于时间线卡片。
3. distributions 用于旁侧年代分布图。

## 6. 错误与兜底策略

1. code=200 视为业务成功。
2. 400 显示“筛选参数异常，请重试”。
3. 404 显示“数据资源不存在”。
4. 500 显示“服务暂时不可用”。
5. 当 overview 或 stats 失败时，页面至少保留空态占位，不阻断主页面导航。

## 7. 联调验收清单

1. overview 返回 summary 与 distributions，字段不缺失。
2. map-points 每个点都有 coordinates.lat/lng。
3. timeline 在筛选前后 matchedBuildings 数量变化正确。
4. stats 的 ratio 保持 0 到 1 之间。
5. 四个接口都遵循统一响应包格式。

## 8. 备注

1. 当前数据源支持 JSON 与 PostgreSQL 双模式。
2. 切换数据源不改变接口路径与响应结构。
3. 如需新增维度，请优先扩展 /api/visualization/stats 的 dimension。
