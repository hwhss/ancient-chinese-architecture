这是一个基于 UniApp + Vue3 + Node.js + Express 的古建筑智能导览系统，融合 AI 问答、建筑知识库、可视化与 3D 模型浏览能力，在文化传播场景中提供“可问、可看、可探索”的数字化体验。

## 4. 可确认的项目事实（请据此生成内容）

### 4.1 技术栈

- 前端：UniApp、Vue3
- 后端：Node.js、Express
- 数据层：JSON 文件数据库 + PostgreSQL（可切换）
- AI能力：关键词检索 + 向量检索（可灰度）+ 大模型兜底
- 3D链路：manifest + 签名URL + Token续期机制

### 4.2 关键后端接口（示例）

- 健康检查：GET /api/health
- AI问答：POST /api/chat
- 建筑列表：GET /api/buildings
- 建筑详情：GET /api/buildings/:id
- 3D信息：GET /api/buildings/:id/model3d
- 3D清单：GET /api/buildings/:id/model3d/manifest
- 素材：GET /api/material, GET /api/materials
- 可视化：/api/visualization/overview, /map-points, /timeline, /stats
- 资产签名：GET /api/assets/signed, GET /api/assets/refresh

### 4.3 当前明确的工程特性

1. 双数据源架构：通过 DATA_SOURCE 支持 json/postgres 切换。
2. Chat检索策略：强规则优先 + 校准重排 + 拒绝机制，降低误召回。
3. 3D资源安全：禁止裸资源外泄，统一签名链路，支持续期与权限绑定。
4. 前端体验优化：骨架屏、消息搜索、收藏、地图聚合、页面动效。
5. 可视化接口增强：overview/map-points/timeline/stats + 兼容别名。

### 4.4 项目结构（摘要）

- frontend/: UniApp前端页面、组件、服务、工具
- backend/: Express服务、路由、控制器、服务层、数据仓储、脚本、迁移
- docs/: 架构、联调、优化、迁移、规范文档
- deploy/: 部署相关资料

### 4.5 示例业务数据（用于演示，不夸大）

- 建筑基础数据规模：17（示例）
- health 接口可返回 datasets 统计、运行模式、时间戳
- Chat 返回包含 source、materialId、可选 debug 字段
