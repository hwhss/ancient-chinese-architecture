# 古建筑项目后端接口对接说明

## 目标

本次调整已完成后端重构，并同步完成前端接口联调代码改造。
当前分支可以直接用于前后端本地联调。

## 后端基础信息

- 本地地址: http://localhost:9527
- 健康检查: GET /api/health
- 兼容测试接口: GET /test
- 返回格式:

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

## 前端统一 API 封装

- 统一请求文件: frontend/services/api.js
- 页面不再直接写 uni.request，而是调用封装方法:
  - chat(question)
  - getBuildings(params)
  - getBuildingById(id)
  - getBuildingModel3d(id)
  - getBuildingModel3dManifest(id)
  - refreshSignedAssetToken(token)
  - getMaterialById(materialId)
  - healthCheck()
  - getApiBaseUrl()
  - setApiBaseUrl(url)
  - clearApiBaseUrl()

### API 地址配置方式

默认地址:

- http://localhost:9527

可选方式 1（构建时环境变量）:

- UNI_APP_API_BASE_URL
- VUE_APP_API_BASE_URL

可选方式 2（运行时覆盖）:

- 调用 setApiBaseUrl("http://你的后端地址:端口")
- 会持久化到本地缓存键 API_BASE_URL

可选方式 3（页面配置）:

- 首页点击“开发设置”进入页面：pages/dev-settings/dev-settings
- 在页面输入后端地址后点击“保存地址”
- 可点击“测试连接”验证当前地址是否可用
- “测试连接”会调用 GET /api/health，适合快速判断服务是否在线
- 可点击“恢复默认地址”清除本地覆盖并回到“环境变量或默认地址”
- 支持“本地/测试/生产”一键切换预设地址
- 测试与生产预设会保存到本地缓存:
  - API_PRESET_TEST
  - API_PRESET_PROD
- 测试与生产预设也支持环境变量默认值:
  - UNI_APP_API_PRESET_TEST / VUE_APP_API_PRESET_TEST
  - UNI_APP_API_PRESET_PROD / VUE_APP_API_PRESET_PROD

好处:

- 接口地址、状态码处理、错误提示策略统一
- 页面代码更简洁，后续更换后端地址或鉴权方式只需改一个文件

## 已提供接口

### 1) 问答接口

- 方法: POST
- 路径: /api/chat
- 请求体:

```json
{
  "question": "太和殿的历史是什么？"
}
```

- 成功响应:

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "question": "太和殿的历史是什么？",
    "answer": "...",
    "source": "knowledge_base",
    "materialId": "gugong_01"
  }
}
```

说明:

- source=knowledge_base 表示命中本地知识库
- source=ai 表示走大模型兜底
- materialId 可能为空，前端需要兜底处理

- 方法: GET
- 路径: /api/buildings/:id/model3d
- 示例: /api/buildings/gugong_01/model3d
- 说明: 用于“查看3D”按钮点击后再请求，避免详情页首屏加载过重

### 7) 建筑 3D 场景清单接口（Manifest）

- 方法: GET
- 路径: /api/buildings/:id/model3d/manifest
- 示例: /api/buildings/gugong_01/model3d/manifest
- 主要字段:
  - activeVersion
  - versions[]: { version, label, format, signedUrl, expiresAt, draco, ktx2, hotspots[] }
  - tokenPolicy: { expiresInSeconds, refreshWindowSeconds, refreshEndpoint, refreshSuggestion }
  - preload: { placeholderType, lod0ModelUrl, panoramaUrl, posterBlur, swapMode, uiHint }
  - scene: { format, entry, environment, draco, ktx2, lod[], expiresAt }
  - camera: { orbit, target, fov }
  - hotspots: []
  - hotspotGuide[]: { id, title, narration }
  - canView / status / poster

- 前端调用建议:
  1. 先请求 /api/buildings/:id 获取概览信息。
  2. 点击“查看3D”后请求 /api/buildings/:id/model3d 判断是否可看。
  3. canView=true 再请求 /api/buildings/:id/model3d/manifest 并加载 scene.entry。

### 8) 3D 模型如何获得（建议流程）

常见来源:

- 实景扫描（摄影测量）:
  - 工具: RealityCapture / Metashape / Polycam / Luma
  - 适合: 遗址、构件、室外场景
- 传统建模:
  - 工具: Blender / 3ds Max / Maya / SketchUp
  - 适合: 复原性建模、历史推演版本
- 公共资源（原型阶段）:
  - Sketchfab / Poly Haven / 开源馆藏资源
  - 注意版权和商用许可

推荐生产链路:

1. 源模型制作（OBJ/FBX/Blend）。
2. 导出 glb/gltf。
3. 压缩优化:
   - 网格压缩: Draco
   - 贴图压缩: KTX2/Basis
4. 上传对象存储/CDN（阿里云 OSS / 腾讯 COS / 七牛 / S3）。
5. 将 modelUrl、poster、status 写入 backend/data/building_profiles.json。
6. 前端通过 manifest 接口按需加载。

### 9) 资源权限签名 URL 机制

- 新增接口: GET /api/assets/signed?token=xxx
- 新增接口: GET /api/assets/refresh?token=xxx
- 作用: 前端不直接拿裸 modelUrl，而是使用后端签名后的短期 URL。
- 签名载荷包含:
  - resourceUrl
  - buildingId
  - version
  - userId / role
  - bindIp / bindUaHash
  - exp(过期时间)

可配置环境变量:

- PUBLIC_BASE_URL: 对外可访问的后端域名（例如 https://api.example.com）
- ASSET_SIGN_SECRET: 签名密钥（生产环境必须修改）
- ASSET_SIGN_EXPIRE_SECONDS: 链接有效期秒数，默认 600
- ASSET_REFRESH_WINDOW_SECONDS: 允许续期窗口秒数，默认 60
- ASSET_REDIRECT_CACHE_SECONDS: 302 浏览器缓存秒数，默认 30
- ASSET_REDIRECT_SHARED_CACHE_SECONDS: 302 边缘缓存秒数，默认 60

流程:

1. 前端请求 /api/buildings/:id/model3d/manifest。
2. 后端返回每个版本的 signedUrl（指向 /api/assets/signed）。
3. 前端 viewer 加载 scene.entry（即 signedUrl）。
4. 后端校验 token 后 302 重定向到真实资源地址。

版本权限说明:

- manifest 内不同 version 可配置 allowedRoles（例如 viewer/admin）。
- 请求 /api/buildings/:id/model3d 或 /manifest 时，后端会按请求角色过滤可见版本。
- 默认从请求头读取角色:
  - x-user-role（默认 viewer）
  - x-user-id（默认 anonymous）

签名绑定说明:

- token 内包含 userId、role、ip、uaHash、exp。
- /api/assets/signed 校验签名与过期时间，并校验 IP/UA 绑定（有值时生效）。
- /api/assets/refresh 仅允许在 refreshWindow 内静默续期；若剩余时间还很长，会返回 shouldRefresh=false。

Token 续期建议（前端）:

1. viewer 加载 scene.entry 后，解析当前版本 expiresAt。
2. 在 expiresAt 前 60 秒触发一次静默续期请求 `/api/assets/refresh`。
3. 若返回 shouldRefresh=true，则用新的 signedUrl 替换后续懒加载资源请求地址。
4. 若返回 shouldRefresh=false，继续使用原 token。
5. 可直接复用 frontend/services/model3dTokenManager.js 中的 createModel3dTokenRefresher。

示例（伪代码）:

```js
const manifest = await getBuildingModel3dManifest(id);
const token = new URL(manifest.scene.entry).searchParams.get("token");
const expiresAtMs = Date.parse(manifest.scene.expiresAt);

setTimeout(
  async () => {
    const refreshed = await refreshSignedAssetToken(token);
    if (refreshed.shouldRefresh) {
      currentSceneEntry = refreshed.signedUrl;
    }
  },
  Math.max(0, expiresAtMs - Date.now() - 60 * 1000),
);
```

模型预加载与加载态建议:

- 优先使用 manifest.preload.placeholderType 决定占位策略:
  - lod0-model: 先加载 lod0ModelUrl 轻量模型
  - panorama: 先显示全景图
  - poster: 退化为静态海报
- 高清模型 scene.entry 加载完成后按 preload.swapMode 无缝替换。
- 可以在 UI 上用 preload.uiHint 作为加载提示文案。

302 缓存策略说明:

- `/api/assets/signed` 成功响应会返回 `Cache-Control: public, max-age, s-maxage`。
- 目的: 减少重复鉴权与 302 计算，降低鉴权服务在高并发浏览下的压力。
- 建议网关/CDN开启按完整 URL 缓存 302，并把 TTL 设置为不超过 token 剩余有效期。

```js
const detail = await getBuildingById(id);
const modelMeta = await getBuildingModel3d(id);

if (modelMeta.canView) {
  const manifest = await getBuildingModel3dManifest(id);
  // viewer.load(manifest.scene.entry)

  // 显示“3D模型准备中”

```

### 11) 展示 3D 模型完整流程（后端到前端）

1. 用户进入详情页，前端请求 /api/buildings/:id 获取文字与概览。
2. 用户点击“查看3D”，前端请求 /api/buildings/:id/model3d。
3. 后端根据 x-user-role 过滤可见版本并返回版本摘要。
4. 前端再请求 /api/buildings/:id/model3d/manifest 获取当前版本 scene.entry（签名URL）。
5. 前端把 scene.entry 传给 3D 查看器（如 model-viewer / three.js）。
6. 查看器发起 GET /api/assets/signed?token=xxx。
7. 后端验签（exp、IP、UA）通过后 302 到真实模型URL。
8. CDN/对象存储返回 glb 资源，前端完成加载并叠加热点讲解。
9. 若用户停留时间较长，在 token 即将过期时调用 `/api/assets/refresh` 静默续期。
10. 对于懒加载高清贴图/子模型，优先使用续期后的 signedUrl，避免“逛久后加载失败”。

## 错误码约定

- 200: 成功
- 400: 参数错误
- 404: 资源不存在
- 500: 服务异常
