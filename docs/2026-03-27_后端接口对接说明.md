# 古建筑项目后端接口对接说明

## 目标

本次调整已完成后端重构，并同步完成前端接口联调代码改造。
当前分支可以直接用于前后端本地联调。

## 后端基础信息

- 本地地址: http://localhost:9527
- 健康检查: GET /api/health
- 兼容测试接口: GET /test
- 返回格式:

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

## 前端统一 API 封装

- 统一请求文件: frontend/services/api.js
- 页面不再直接写 uni.request，而是调用封装方法:
  - chat(question)
  - getBuildings(params)
  - getBuildingById(id)
  - getMaterialById(materialId)
  - healthCheck()
  - getApiBaseUrl()
  - setApiBaseUrl(url)
  - clearApiBaseUrl()

### API 地址配置方式

默认地址:

- http://localhost:9527

可选方式 1（构建时环境变量）:

- UNI_APP_API_BASE_URL
- VUE_APP_API_BASE_URL

可选方式 2（运行时覆盖）:

- 调用 setApiBaseUrl("http://你的后端地址:端口")
- 会持久化到本地缓存键 API_BASE_URL

可选方式 3（页面配置）:

- 首页点击“开发设置”进入页面：pages/dev-settings/dev-settings
- 在页面输入后端地址后点击“保存地址”
- 可点击“测试连接”验证当前地址是否可用
- “测试连接”会调用 GET /api/health，适合快速判断服务是否在线
- 可点击“恢复默认地址”清除本地覆盖并回到“环境变量或默认地址”
- 支持“本地/测试/生产”一键切换预设地址
- 测试与生产预设会保存到本地缓存:
  - API_PRESET_TEST
  - API_PRESET_PROD
- 测试与生产预设也支持环境变量默认值:
  - UNI_APP_API_PRESET_TEST / VUE_APP_API_PRESET_TEST
  - UNI_APP_API_PRESET_PROD / VUE_APP_API_PRESET_PROD

好处:

- 接口地址、状态码处理、错误提示策略统一
- 页面代码更简洁，后续更换后端地址或鉴权方式只需改一个文件

## 已提供接口

### 1) 问答接口

- 方法: POST
- 路径: /api/chat
- 请求体:

```json
{
  "question": "太和殿的历史是什么？"
}
```

- 成功响应:

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "question": "太和殿的历史是什么？",
    "answer": "...",
    "source": "knowledge_base",
    "materialId": "gugong_01"
  }
}
```

说明:

- source=knowledge_base 表示命中本地知识库
- source=ai 表示走大模型兜底
- materialId 可能为空，前端需要兜底处理

### 2) 单个素材接口

- 方法: GET
- 路径: /api/material?materialId=gugong_01

- 成功响应:

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "url": "https://...",
    "type": "image",
    "source": "故宫博物院官网",
    "materialId": "gugong_01"
  }
}
```

### 3) 素材列表接口

- 方法: GET
- 路径: /api/materials
- 可选查询: type=image|video

### 4) 建筑列表接口

- 方法: GET
- 路径: /api/buildings
- 可选查询:
  - category: palace|bridge|garden|defense|residence|tower|water|all
  - q: 搜索关键词

- 成功响应 data 为建筑数组，字段包含:
  - id
  - name
  - category
  - location
  - description
  - image
  - tags

### 5) 建筑详情接口

- 方法: GET
- 路径: /api/buildings/:id
- 示例: /api/buildings/gugong_01

### 6) 知识库接口

- 方法: GET
- 路径: /api/knowledge

## 前端改造完成项

1. map 页已改为使用 /api/buildings 拉取列表，并保留分类筛选。
2. chat 页已改为优先使用接口返回的 materialId，仅保留关键词映射作降级兜底。
3. detail 页已改为先查 /api/buildings/:id 再查 /api/material，并在失败时展示占位素材。
4. 错误提示已按 code 和 HTTP 状态码做基础兜底。

## 错误码约定

- 200: 成功
- 400: 参数错误
- 404: 资源不存在
- 500: 服务异常

## 后端数据源

- backend/data/knowledge_base.json
- backend/data/material_links.json
- backend/data/buildings.json

后续若接数据库，保持接口不变即可平滑迁移。
