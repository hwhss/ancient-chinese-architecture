# 后端接口总览与对接手册

## 1. 文档定位

本文件是前后端联调的唯一接口总文档（长期维护版，无日期）。

适用范围：

1. 前端页面联调与功能开发。
2. 可视化页面数据对接。
3. 3D 资源签名链路联调。
4. AI 问答与检索链路联调。

说明：

1. 历史日期文档用于记录阶段性变更。
2. 日常开发以本文件为准。

---

## 2. 基础信息

1. 默认本地地址： http://localhost:9527
2. 健康检查： GET /api/health
3. 统一成功响应格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

4. 统一错误响应格式：

```json
{
  "code": 400,
  "msg": "错误说明",
  "error": "可选错误详情"
}
```

---

## 3. 鉴权与请求头约定

当前后端不要求登录态 token，但部分接口支持以下请求头用于权限与签名绑定：

1. x-user-id：用户标识（默认 anonymous）
2. x-user-role：用户角色（默认 viewer）
3. User-Agent：用于签名绑定校验
4. X-Forwarded-Proto：反向代理下协议识别（manifest 签名地址生成）

3D 相关接口建议前端在联调时固定传递 x-user-id 和 x-user-role，便于后续权限扩展。

## 6. Chat 问答接口

## 6.1 POST /api/chat

请求体：

```json
{
  "question": "故宫最大那个殿它的高度是多少啊？"
}
```

成功 data：

```json
{
  "question": "故宫最大那个殿它的高度是多少啊？",
  "answer": "...",
  "source": "knowledge_base",
  "materialId": "gugong_01",
  "debug": {
    "rawQuestion": "...",
    "retrievalQuestion": "...",
    "usedNormalizedQuestion": true,
    "vectorRetrievalEnabled": true,
    "dataSource": "postgres",
    "matchedBy": "knowledge_hybrid",
    "rewriteApplied": true,
    "fallbackToRawQuestion": false,
    "matchScore": {
      "keywordScore": 20,
      "vectorSimilarity": 0.72,
      "hybridScore": 0.48
    }
  }
}
```

说明：

1. source 可能值：knowledge_base、knowledge_hybrid、knowledge_vector、ai。
2. debug 字段按配置开关返回（开发建议开启，生产建议关闭）。
3. question 为空会返回 400。
4. 问题长度超过 QUESTION_MAX_LENGTH 会返回 400。

## 7. 素材接口

## 7.1 GET /api/material

Query：

1. materialId（必填）

成功 data：

```json
{
  "url": "https://...",
  "type": "image",
  "source": "故宫博物院官网",
  "materialId": "gugong_01"
}
```

错误：

1. materialId 为空 -> 400
2. material 不存在 -> 404

## 7.2 GET /api/materials

Query：

1. type（可选，例 image 或 video）

成功 data：素材数组。

## 9. 建筑基础接口

## 9.1 GET /api/buildings

Query（可选）：

1. category： palace|bridge|garden|defense|residence|tower|water|all
2. q：关键词搜索

成功 data：建筑摘要数组，字段包括：

1. id, name, category, categoryText
2. location, province, city, coordinates
3. heritageLevel, openStatus, mainEra
4. coverImage, tags, overviewSummary
5. has3d, model3dStatus

## 9.2 GET /api/buildings/:id

成功 data：建筑详情对象，字段包括：

1. 基础字段：id/name/category/location 等
2. overview：summary/era/style/keyPoints
3. history, architectureHighlights, culturalValue
4. model3d：status/canView/viewerType/modelUrl/poster/note

不存在返回 404。

---

## 10. 3D 相关接口

## 10.1 GET /api/buildings/:id/model3d

用途：返回某建筑的 3D 可看状态与版本摘要。

成功 data 重点字段：

1. buildingId, buildingName
2. status, canView, viewerType
3. activeVersion
4. versions[]：version/label/format/signedUrl/expiresAt

## 10.2 GET /api/buildings/:id/model3d/manifest

用途：返回前端 viewer 加载所需完整场景清单。

成功 data 重点字段：

1. buildingId, buildingName, status, canView, poster
2. activeVersion
3. versions[]：version/label/format/signedUrl/expiresAt/draco/ktx2
4. tokenPolicy：expiresInSeconds/refreshWindowSeconds/refreshEndpoint/refreshSuggestion
5. preload：placeholderType/lod0ModelUrl/lod0ExpiresAt/panoramaUrl/posterBlur/swapMode/uiHint
6. scene：format/entry/environment/draco/ktx2/lod[]/expiresAt
7. camera
8. hotspots
9. hotspotGuide
10. note

### 10.3 GET /api/assets/signed

Query：

1. token（必填）

行为：

1. 校验 token（签名/过期/IP/UA）。
2. 通过后 302 重定向到真实资源 URL。

### 10.4 GET /api/assets/refresh

Query：

1. token（必填）

成功 data：

```json
{
  "shouldRefresh": true,
  "secondsLeft": 30,
  "signedUrl": "https://.../api/assets/signed?token=...",
  "expiresAt": "2026-03-30T12:10:00.000Z",
  "token": "..."
}
```

说明：

1. 若 token 仍很新，返回 shouldRefresh=false。
2. 若可续期，返回 shouldRefresh=true 和新 token。

## 11. 可视化接口（聚合）

统一前缀： /api/visualization

兼容别名：

1. /api/visual/info-graphic -> timeline
2. /api/visual/chart -> stats

## 11.1 GET /api/visualization/overview

用途：首页总览 KPI。

成功 data 重点字段：

1. type（overview）
2. summary：totalBuildings/withCoordinates/with3d/coverageRatio
3. distributions：byCategory/byRegion/byHeritageLevel/byEraBucket
4. kpis[]：可直接渲染卡片
5. updatedAt

## 11.2 GET /api/visualization/map-points

用途：地图点位。

成功 data 重点字段：

1. type（map-points）
2. points[]：点位主数组
3. markers：points 的兼容别名
4. geoJson：FeatureCollection（地图组件可直接使用）
5. points[].model3d：status/viewerType/modelUrl/poster
6. meta：totalBuildings/returnedPoints/missingCoordinates

## 11.3 GET /api/visualization/timeline

用途：时间轴与信息图。

Query（可选）：

1. category
2. region（或 province）
3. city
4. eraStart
5. eraEnd

成功 data 重点字段：

1. type（timeline）
2. filters
3. timeline[]
4. distributions
5. infographic：type/title/layout/nodes/edges/labels/links
6. meta, updatedAt

## 11.4 GET /api/visualization/stats

用途：图表统计。

Query（可选）：

1. dimension：category|era|region|heritageLevel|openStatus（默认 category）
2. category
3. region（或 province）
4. city
5. eraStart
6. eraEnd

成功 data 重点字段：

1. type（chart）
2. filters
3. dimension
4. items[]：key/count/ratio
5. chart：title/unit/xAxis/series/legend/tooltip
6. meta, updatedAt

---

## 12. 前端建议调用顺序

1. 应用启动先调 /api/health，展示运行态。
2. 首页并行调 /api/visualization/overview + /api/visualization/stats。
3. 地图页调 /api/visualization/map-points。
4. 时间轴页调 /api/visualization/timeline。
5. 聊天页调 /api/chat，必要时根据 materialId 联动素材展示。
6. 3D 查看时按顺序调用：
   - /api/buildings/:id
   - /api/buildings/:id/model3d
   - /api/buildings/:id/model3d/manifest

---

## 13. 常见错误与排查

1. chat 始终 ai：检查 runtime.vectorRetrievalEnabled、向量配置、阈值。
2. signed 链接 401：检查 token 是否过期，x-user-id/x-user-role/IP/UA 是否变化。
3. map-points 缺点位：检查 building 的 lat/lng。
4. postgres 模式报连接问题：检查 DATABASE_URL 与数据库凭据。
