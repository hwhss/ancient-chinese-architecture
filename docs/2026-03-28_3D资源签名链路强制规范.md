# 3D资源签名链路强制规范（防外泄与联调故障）

## 1. 目的

本规范用于避免以下常见问题：

1. 真实模型地址（CDN/对象存储）在接口中外泄。
2. 真机联调或代理访问时，签名地址指向 localhost 导致不可达。
3. 历史数据字段不一致（modelUrl/url）导致部分 LOD 漏签。

适用范围：

- 后端 3D Manifest 相关接口
- 资产签名与续期接口
- 前端 three.js viewer 对接

## 2. 强制要求（必须修改/必须遵守）

### 2.1 Manifest 禁止返回裸资源地址

必须：

1. /api/buildings/:id/model3d/manifest 的 versions[] 仅返回最小必要字段：
   - version
   - label
   - format
   - signedUrl
   - expiresAt
   - draco
   - ktx2
2. scene.entry、scene.lod[].modelUrl/url、preload.lod0ModelUrl 必须为签名地址。

禁止：

1. 在 manifest 中返回任意裸 modelUrl（例如 https://cdn.xxx/model.glb）。
2. 在 versions[] 中透传原始模型地址、存储桶地址、临时下载直链。

### 2.2 签名 URL 生成禁止写死 localhost

必须：

1. 签名 URL 生成优先使用请求域名（request host/proto）。
2. 请求域名不可用时，才兜底使用 PUBLIC_BASE_URL。
3. 反向代理场景需支持 x-forwarded-proto。

禁止：

1. 在服务代码里写死 http://localhost:9527 作为签名 URL 基址。
2. 假设前端、后端总在同一台机器。

### 2.3 LOD 字段兼容处理（防漏签）

必须：

1. 后端签名逻辑同时兼容读取 item.modelUrl 和 item.url。
2. 输出时统一回填为签名地址（modelUrl/url/signedUrl 至少保证前端读取的字段是签名地址）。

禁止：

1. 仅处理 modelUrl 而忽略 url。
2. 对空值不做防御，导致部分 LOD 未签名直接透传。

### 2.4 Token 与权限绑定

必须：

1. token payload 至少包含：resourceUrl、buildingId、version、userId、role、exp。
2. 如已启用环境绑定，校验 IP 与 UA 哈希。
3. refresh 接口必须校验 token 所属 userId/role 与当前请求一致。

禁止：

1. refresh 时不校验所属用户/角色。
2. 使用长期不过期签名替代短期签名+续期。

## 3. 环境变量强制清单

生产环境必须设置：

1. PUBLIC_BASE_URL
2. ASSET_SIGN_SECRET（必须替换默认值）
3. ASSET_SIGN_EXPIRE_SECONDS
4. ASSET_REFRESH_WINDOW_SECONDS
5. ASSET_REDIRECT_CACHE_SECONDS
6. ASSET_REDIRECT_SHARED_CACHE_SECONDS

建议值（示例）：

1. ASSET_SIGN_EXPIRE_SECONDS=600
2. ASSET_REFRESH_WINDOW_SECONDS=60
3. ASSET_REDIRECT_CACHE_SECONDS=30
4. ASSET_REDIRECT_SHARED_CACHE_SECONDS=60

禁止：

1. 生产环境使用默认 dev secret。
2. 未配置 PUBLIC_BASE_URL 且网关未透传 host/proto。

## 4. 前端对接强制项（避免会话中断）

必须：

1. viewer 在 manifest 拉取后启动 token refresher。
2. 页面卸载时停止 refresher。
3. 续期成功后更新 manifest.scene.entry 与 manifest.scene.expiresAt。
4. 高清替换失败保留 LOD0，不允许白屏。

禁止：

1. 把 scene.entry 当成永久可用地址。
2. 失败后无限重试，不做次数上限。

## 5. 上线前检查清单（逐条打勾）

1. 抓包检查 manifest，确认 versions[] 无裸 modelUrl。
2. 检查 scene.entry、scene.lod[]、preload.lod0ModelUrl 均为 /api/assets/signed?token=...。
3. 在真机/非 localhost 域名下联调，确认签名地址域名正确可达。
4. 用含 url 字段的历史数据验证 LOD 仍会签名。
5. refresh 在窗口外返回 shouldRefresh=false，在窗口内返回 shouldRefresh=true。
6. refresh 后新的 signedUrl 可继续加载模型资源。
7. 修改 x-user-id 或 x-user-role 后，旧 token 刷新应被拒绝。
8. ASSET_SIGN_SECRET 非默认值。

## 6. 故障定位速查

1. 现象：模型一直加载失败（真机）
   - 优先检查 signedUrl 域名是否是 localhost。
2. 现象：某些 LOD 能加载，某些不行
   - 优先检查历史数据是否使用 url 字段且被漏签。
3. 现象：逛一段时间后加载失败
   - 优先检查前端是否接入 refresher 和续期后地址替换。
4. 现象：资源被越权访问
   - 优先检查 manifest 是否泄露裸地址、refresh 是否校验 userId/role。

## 7. 变更原则

1. 任何新增模型字段若可直达真实资源，必须走签名化。
2. 任何新增环境下的访问域名变化，必须先验证签名 URL 生成策略。
3. 任何续期机制改动，必须同时覆盖主模型与 LOD 路径。
